# Impeccable — Design Craft Skill

Design-craft delegate from [impeccable by pbakaus](https://github.com/pbakaus/impeccable).

## Use This Skill When

- Designing or critiquing any frontend interface
- Polishing visual layout, spacing, typography
- Improving a component's visual hierarchy
- Implementing animation and motion design
- Audit design system consistency

## Motion Decision Framework

From Emil Kowalski — when to animate, what easing, how fast:

1. **Purpose** — Does the animation serve a function (feedback, orientation, guidance)? If not, skip it.
2. **Timing** — Fast interactions (100-200ms), moderate (200-350ms), dramatic (350-500ms max). Never exceed 500ms.
3. **Easing** — Use cubic-bezier curves, not linear. Eases should start fast and end smooth.
4. **Scope** — Animate the smallest change possible. Don't animate a whole section when a single element needs feedback.

## Craft Checklist

- Consistent spacing (4px/8px grid)
- Clear visual hierarchy (size, weight, color)
- Proper contrast ratios
- Consistent corner radius
- Consistent shadow elevation
- State variations (hover, focus, active, disabled)
